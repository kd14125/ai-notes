## Codex 最基础的用法

Codex 和普通聊天工具的区别，在于它可以进入你指定的文件夹，读取文件、修改内容并运行本地工具。因此，第一次交代任务时最好说清楚四件事：

1. 要做什么。
2. 相关文件在哪里。
3. 哪些内容不能改。
4. 做完以后怎样检查。

例如：

```text
请检查 D:/论文/初稿 中的 Markdown 文档。
只修正错别字和明显病句，不要修改公式、数据和参考文献。
修改前先列出发现的问题，得到我的确认后再动文件。
完成后告诉我改了哪些文件。
```

任务很简单时可以直接让 Codex 执行。涉及整篇论文、多个目录或大量文件时，建议先使用计划模式：

```text
/plan 请先检查当前论文项目的目录、模板和参考文献，给出整理方案，暂时不要修改文件。
```

更完整的用法见 [Codex 进阶使用](./2.codex使用进阶.md)。

## 用 AI 阅读论文

我目前比较常用的思路是：先把 PDF 转成便于检索的 Markdown，再让 AI 结合原文做总结，最后把论文和笔记放回 Zotero 管理。

一个常见流程如下：

1. 使用 MinerU 解析 PDF，保留正文、公式、表格和图片。
2. 让 AI 按研究问题、方法、数据集、实验结果和局限整理笔记。
3. 对重要结论回到原始 PDF，核对页码、图表和上下文。
4. 将 PDF、Markdown 和阅读笔记关联到同一个 Zotero 条目。

可以从这个提示开始：

```text
请阅读这篇论文，先不要泛泛总结。
重点整理研究问题、方法、数据集、评价指标、主要结果和作者承认的局限。
每个重要结论都标出原文位置；找不到依据时明确写“未在原文中确认”。
最后说明它与我的研究主题有哪些直接关系。
```

## 用 AI 辅助写论文

AI 更适合参与具体环节，而不是一次生成整篇论文。可以让它帮助梳理选题、整理文献证据、检查章节逻辑、规划实验或修改语言，但引用、数据和结论必须由作者负责。

建议按照这个顺序使用：

| 阶段 | 可以交给 AI 的工作 | 需要自己确认的内容 |
| --- | --- | --- |
| 选题与设计 | 拆分研究问题、比较方法、整理实验方案 | 研究价值、条件是否可行 |
| 文献阅读 | 分类文献、提取方法和实验信息 | 原文含义、引用是否准确 |
| 初稿写作 | 根据已有证据组织段落、检查逻辑 | 观点、数据、专业判断 |
| 实验与制图 | 生成分析代码、图表和可编辑示意图 | 原始数据、统计方法、可复现性 |
| 修改与投稿 | 语言修改、格式检查、模拟审稿 | 最终表述、作者责任、期刊要求 |

仓库中的 `research-writing-skill`、`MinerU`、`Zotero`、`writing-humanizer-zh` 等工具各有分工。建议按当前问题选择，不必一次安装全部 Skills。

## 使用时注意

- 不要把 API Key、Token、Cookie、账号密码或未公开数据发给 AI，也不要提交到公开仓库。
- AI 生成的 DOI、作者、页码和参考文献可能是错的，正式引用前必须回到原文核对。
- AI 可以规划实验，不能替代真实实验。论文中的数据、图表和结论要能够复现。
- 修改重要文件前先备份，或者要求 Codex 保留原文件并标出修改位置。
- 第三方 API、代理工具和免费额度可能发生变化，请以各自官网的最新说明为准。
